# RAG: Retrieval-Augmented Generation · 檢索增強生成

## Key Terms

  data={[
    {
      term: "RAG · 檢索增強生成",

        "A pattern where a model answers using retrieved documents plus the user query, instead of relying only on weights baked in at training. 先從知識庫檢索相關片段，再連同問題一併交給語言模型生成答案，而非只靠訓練時「記住」的內容。",

    },
    {
      term: "Embedding · 向量嵌入",
      meaning:

    },
    {
      term: "Vector store · 向量庫",
      meaning:
        "Index of document chunks and their embeddings for fast nearest-neighbour search. 儲存文件塊與對應向量，支援快速近鄰搜尋的索引／資料庫。",
    },
    {
      term: "Chunking · 分塊",
      meaning:
        "Splitting documents into smaller pieces (with optional overlap) so retrieval stays focused and fits context limits. 把長文件切成較小單位（可含重疊），令檢索更精準並符合模型上下文長度。",

    {

      meaning:
        "Answering with explicit support from retrieved passages, reducing pure fabrication. 答案應可追溯到檢索到的段落，降低無中生有（幻覺）的機會。",
    },
  ]}

  caption="End-to-end RAG flow · RAG 端到端流程"

    },
    {
      step: "5",
      action:

    {
      step: "6",
      action:

  ]}

## RAG vs fine-tuning · RAG 與微調

<BilingualParagraph
  en="Fine-tuning changes model behaviour and style by updating weights; RAG changes facts available at inference. They complement each other: RAG for factual grounding, fine-tuning for tone, format, or domain phrasing."
  zh="微調透過更新權重改變模型行為與語氣；RAG 則在推理時改變「手頭有甚麼資料」。兩者常並用：RAG 負責事實與依據，微調負責語氣、格式或行業用語習慣。"
/>
